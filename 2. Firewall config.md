After creating the basic elements needed to start the lab, we will need to configure the virtual machine's firewall to make it accessible to attackers.

Under no circumstances should this configuration be done on a machine that does require protection, whether in a corporate or personal environment. This machine will be completely exposed for the purpose of the lab, which is to collect information about attackers and lure them into attempting to access the machine.

From the list of resources, we will see an item called <VM_name>-nsg, which corresponds to the firewall. Upon accessing it, we will find several pre-created rules for incoming and outgoing traffic.

![imagen](https://github.com/user-attachments/assets/e9e55166-531f-4363-95b8-dcb35becbfc1)

We will remove the first rule created for incoming traffic (RDP) so that any type of connection, not just RDP, can be made from anywhere in the world without restriction. Below are the details of how it is configured.

![imagen](https://github.com/user-attachments/assets/f22ab73f-a29a-4e30-8d6a-072b887d986e)

Because this configuration makes the machine vulnerable, we will see several disclaimers indicating that the machine will be unprotected and completely exposed, and that this poses a risk.

![imagen](https://github.com/user-attachments/assets/b6ef3464-96aa-45e2-8c06-c1fa0e8790fc)

Once correctly configured, it will appear in the firewall rules list.

![imagen](https://github.com/user-attachments/assets/916ae017-8433-45c9-ad30-987bd651b8c5)

Last but not least, we are going to log into the VM and disable the internal firewall. In the virtual machine information, we have the public IP address that we can use to access it:

![imnagen](https://github.com/user-attachments/assets/b3e34491-f66e-4f79-8ce0-26becdd307dd)

We will use the credentials configured when creating the virtual machine and the RDP service. Inside the machine, we will carry out two steps.
1) The first thing we will see upon entering is this device security settings screen, where we will disable all options.
![imagen](https://github.com/user-attachments/assets/bde09053-fd0a-44ec-9d47-6205bbbb400a)
2) Once on the Desktop, type "wf.msc" in the search bar to open the Windows firewall and configure it.
![imagen](https://github.com/user-attachments/assets/6a667f7b-7927-472c-b6ca-21d746499117)
When you access "Windows Defender Firewall Properties", a window will pop up where you can switch the firewall status to Off (Domain Profile, Private Profile, Public Profile).
![imagen](https://github.com/user-attachments/assets/9cbb9cb3-22bc-4ebb-bb2b-7625a408ea87)
