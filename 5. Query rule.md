To create the detection rule for multiple failed login attempts on our virtual machine, we will create a new schedule rule named “Multiple login failures” with the description “In the last 1 hour, multiple failed login attempts were detected from the same source IP address. This may be an indication of a brute force attack.” In my case, I set the severity to Medium and selected Technique T1110 (Credential Access - Brute Force). 

I configured the rule so that I could run tests without having to wait too long, but the thresholds can be adjusted to better suit the organization's needs. For that reason, I used the logs from the past hour and set the threshold to 5 failed attempts.

```
SecurityEvent
| where EventID == 4625
| where TimeGenerated > ago(1h)
| summarize Attempts = count(), Timestamp = max(TimeGenerated) by IpAddress, TargetAccount
| where Attempts > 5
| order by Timestamp desc
```
